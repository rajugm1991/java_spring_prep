## üìñ Problem Format

Each problem includes:

1. **Problem Statement**: Clear description of the problem
2. **Requirements**: Functional and non-functional requirements
3. **Constraints**: Scale, latency, availability requirements
4. **Solution Approach**: Step-by-step design process
5. **Architecture Diagram**: Visual representation
6. **Code Examples**: Implementation snippets
7. **Trade-offs**: Pros and cons of design choices
8. **Alternatives**: Other possible approaches
9. **Interview Tips**: What interviewers look for
10. **Follow-up Questions**: Common follow-ups
