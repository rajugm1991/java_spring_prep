## Performance

### Q1: How do you measure application performance?

**Answer:**
- **Latency**: Response time (p50, p95, p99)
- **Throughput**: Requests per second
- **Error Rate**: Percentage of failed requests
- **Resource Usage**: CPU, memory, network

---

### Q2: How do you optimize database queries?

**Answer:**
- **Indexes**: Proper indexing strategy
- **Query Optimization**: Avoid N+1 queries
- **Connection Pooling**: Reuse connections
- **Caching**: Cache frequently accessed data

## Scalability

### Q3: What is the difference between horizontal and vertical scaling?

**Answer:**
- **Vertical**: Add resources to existing machine (CPU, RAM)
- **Horizontal**: Add more machines
- Horizontal is preferred for distributed systems

---

### Q4: How do you scale a database?

**Answer:**
- **Read Replicas**: Scale reads
- **Sharding**: Partition data
- **Caching**: Reduce database load
- **Denormalization**: Trade storage for performance
